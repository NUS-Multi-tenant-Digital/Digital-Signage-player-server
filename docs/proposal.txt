2пёҸ Device Communication Flow (Important)
Your system has two main flows:

рҹ”„ A. Content Sync Flow (Normal Operation)
1.Player boots
2.Authenticates using:
oDevice ID
oDevice token
3.Fetches:
oAssigned layout
oSchedule
oMedia manifest
4.Downloads & caches media locally
5.Starts rendering
Polling frequency:
пӮ·Schedule check: every 1вҖ“5 minutes
пӮ·Heartbeat: every 30вҖ“60 seconds

рҹҡЁ B. Real-Time Push Flow (Optional but Powerful)
Using:
пӮ·WebSocket OR
пӮ·MQTT broker
Example:
Admin updates layout вҶ’
Backend publishes event вҶ’
Player receives event вҶ’
Instant refresh.
This enables:
пӮ·Emergency override
пӮ·Instant promotion update
пӮ·Live announcements
